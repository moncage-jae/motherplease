# Docs

프로젝트 문서를 보관합니다. 전원 열람 가능하며, 기획자가 주로 관리합니다.

## 하위 폴더

| 폴더 | 내용 |
|------|------|
| `GDD/` | 게임 디자인 문서 (기획서, 시스템 설계서, 레벨 디자인 등) |
| `TDD/` | 기술 설계 문서 (아키텍처, API 설계, 기술 스펙 등) |
| `References/` | 레퍼런스 이미지, 영상 링크, 무드보드 |
| `MeetingNotes/` | 회의록 |

## 규칙

- 문서 파일명에 날짜를 포함합니다. (예: `회의록_20260326.md`)
- HTML 기획서는 GDD 폴더에 넣습니다.
- 레퍼런스 이미지는 References 폴더에, 링크 모음은 마크다운으로 정리합니다.
